Define variables

Defines one or more variables in memory. It is said to be variable because its content may vary, but never its name or its type. The definition of a variable forces you to boot with an expression that is compatible with the data type of the variable, the multi-statement has allowed multiple settings and initializations of these settings to be rescaned in a single command.

Syntax 

[define] <TYPE> <var> [= <expression>][, <var> [= <expression>] …]


<TYPE> – Data type (int | real | logic | text)

<var> – Variable name subject to the following rules
 o It can't be a reserved word
 o You must start with a letter (a-z; A-Z)
 o May contain digits (0-9)
 o It should be significant from what stores

[<expression>] - computational expression whose result type is compatible with the variable data type, this is optional because if not inserted the variable will take the default value

Example

set an integer variable with the value 10 
 define integer i = 10
define a real variable ( with the value 0.0 by default) 
 define real r
set two integer variables (with zero value by default) 
 define integer i1,i2
define a varied text (omission of the type) 
 define string txt = "Algorithmi"
