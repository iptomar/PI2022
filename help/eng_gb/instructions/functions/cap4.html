4 Functions

Functions do not function as subs that can be invoked within instructions.


Flowchart EN-GH
Function *type* *name*([*param*,..,*param*])
*instructions*
end name

Type –> type of calculation performed by the function

Name -> Role name subject to the following rules:

  • Cannot be a reserved word or other function
  • Must start with a letter (a-z ; A-Z)
  • May contain digits (0-9)
  • Must be significant of what stores

(*Parameter*) –> Function Parameters (Optional)

  • type –> Parameter data type
  • Variables -> Parameter name

4.1.3 Get a value from a function

Functions can perform calculations that are returned to the location where the function was invoked. The RETURN statement allows returning a value from the function. Whenever this instruction is executed the function ends.

Flowchart GAL
return *expression*
  • *expression* –> Computational expression compatible with the function's return data type
    • If the function returns Empty , the expression must be empty.
